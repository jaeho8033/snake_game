# SPEC-GAME-002 인수 기준 (Acceptance Criteria)

Given-When-Then 형식의 인수 시나리오. 각 요구사항 모듈당 최소 2개 시나리오를 포함하며, 엣지 케이스를 명시한다. 모든 시나리오는 결정론적 단위 테스트(주입된 틱 주기/장애물 셀)로 검증 가능해야 한다.

---

## REQ-EXT-001: 난이도 / 속도 제어

### AC-DIFF-1: ready에서 "높음" 선택 후 시작 → 루프 틱 주기 일치

- **Given**: 게임 상태가 `ready`이고, `difficulty.ts`에 "높음" 프로필의 틱 주기 값이 정의되어 있다.
- **When**: 사용자가 난이도 "높음"을 선택하고 게임을 시작(`playing` 전환)한다.
- **Then**: 게임 루프의 틱 주기가 "높음" 프로필에 정의된 틱 주기 값과 정확히 일치한다.

### AC-DIFF-2: playing 중 난이도 변경 → 현재 게임 유지, 다음 재시작부터 적용

- **Given**: 게임 상태가 `playing`이고, 현재 틱 주기는 "중간" 프로필 값이다.
- **When**: 사용자가 게임 진행 중에 난이도를 "낮음"으로 변경한다.
- **Then**: 현재 진행 중인 게임의 틱 주기는 "중간" 값을 그대로 유지한다.
- **And**: 이후 게임을 재시작(`restart`)하면, 새 게임의 틱 주기가 "낮음" 프로필 값으로 적용된다.

### AC-DIFF-3 (edge): playing 동안 틱 주기 불변

- **Given**: 게임 상태가 `playing`이고 난이도 변경 입력이 전혀 없다.
- **When**: 여러 틱이 경과한다(예: 10틱).
- **Then**: 모든 틱에서 적용된 틱 주기가 선택된 난이도 프로필 값으로 일정하게 유지된다(틱마다 값이 흔들리지 않음).

---

## REQ-EXT-002: 벽 장애물

### AC-OBS-1: 장애물 모드, 머리 앞 셀에 장애물 → 한 틱 후 gameover

- **Given**: 장애물 모드가 활성화되어 있고, 뱀 머리의 진행 방향 바로 앞 셀에 장애물 셀이 배치되어 있으며, 게임 상태는 `playing`이다.
- **When**: 게임이 한 틱 진행되어 뱀 머리가 해당 장애물 셀로 이동을 시도한다.
- **Then**: 시스템은 게임 상태를 `gameover`로 전환한다(벽/자체 충돌과 동일한 충돌 규칙으로 처리).

### AC-OBS-2: 장애물 모드, 새 음식 생성 → 빈 셀에만 생성

- **Given**: 장애물 모드가 활성화되어 있고, 장애물 셀 집합과 뱀 세그먼트가 게임판의 일부 셀을 점유하고 있다.
- **When**: 음식 섭취 등으로 새 음식이 생성된다.
- **Then**: 생성된 음식의 위치는 장애물 셀도 아니고 뱀 세그먼트도 아닌 빈 셀이다(여러 번 생성해도 항상 빈 셀 보장).

### AC-OBS-3 (edge): 장애물 모드로 시작 → 초기 세그먼트와 장애물 비겹침

- **Given**: 장애물 모드가 활성화되어 있고, config에 장애물 셀 집합과 뱀 초기 위치가 정의되어 있다.
- **When**: 게임을 시작한다(`ready` → `playing`).
- **Then**: 뱀의 모든 초기 세그먼트 좌표가 장애물 셀 집합과 하나도 겹치지 않는다(시작 즉시 게임 오버가 발생하지 않음).

---

## 엣지 케이스 요약 (Edge Cases)

- AC-DIFF-2: `playing` 중 변경이 즉시 반영되지 않고 펜딩 후 재시작에 적용되는지 검증.
- AC-DIFF-3: 단일 게임 세션 내 틱 주기 불변성 검증.
- AC-OBS-3: 초기 배치 비겹침으로 시작 즉시 게임 오버 방지.

## 품질 게이트 (Quality Gate)

- 모든 인수 시나리오(AC-DIFF-1~3, AC-OBS-1~3)가 단위 테스트로 통과한다.
- 신규/수정 게임 코어 로직 커버리지 95% 이상(전체 85% 이상).
- 장애물 충돌이 벽/자체 충돌과 단일 규칙으로 통합되어 별도 게임 오버 경로가 없음을 코드 리뷰로 확인.
- SPEC-GAME-001 기존 인수 기준(AC-SCORE/AC-SELF/AC-RENDER 등)이 본 SPEC 변경으로 회귀하지 않음을 확인.
