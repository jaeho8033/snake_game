# 인수 기준 - SPEC-GAME-001: Playable Snake MVP

모든 시나리오는 Given/When/Then 형식이다. 핵심 동작별로 최소 2개 이상의 시나리오를 정의한다.

---

## 1. 뱀 이동 (Movement) — REQ-GAME-001

### AC-MOVE-1: 정상 전진
- **Given** 게임 상태가 `playing`이고 뱀이 오른쪽 방향으로 진행 중이며 머리가 (5, 5)에 있다
- **When** 한 틱이 진행된다(음식 미섭취)
- **Then** 머리는 (6, 5)로 이동하고 뱀의 전체 길이는 변하지 않는다(꼬리 한 셀 제거)

### AC-MOVE-2: 방향 유지
- **Given** 게임 상태가 `playing`이고 뱀이 위쪽으로 진행 중이며 입력이 없다
- **When** 연속 3틱이 진행된다
- **Then** 뱀은 매 틱마다 위쪽으로 한 셀씩 직선 이동한다

---

## 2. 음식 섭취 및 성장 (Eating / Growth) — REQ-GAME-002

### AC-EAT-1: 길이 증가 및 새 음식 생성
- **Given** 뱀의 머리 바로 앞 셀에 음식이 있다
- **When** 한 틱이 진행되어 머리가 음식 셀로 이동한다
- **Then** 뱀의 길이가 1 증가하고(꼬리 미제거), 새 음식이 빈 셀에 생성된다

### AC-EAT-2: 음식은 항상 하나
- **Given** 게임이 `playing` 상태로 진행 중이다
- **When** 음식을 섭취하여 새 음식이 생성된다
- **Then** 게임판 위 음식의 개수는 항상 정확히 1개를 유지한다

---

## 3. 벽 충돌 (Wall Collision) — REQ-GAME-003

### AC-WALL-1: 우측 경계 충돌
- **Given** 뱀의 머리가 게임판 우측 끝 열에 있고 오른쪽으로 진행 중이다
- **When** 한 틱이 진행된다
- **Then** 게임 상태가 `gameover`로 전환되고 뱀은 경계 밖으로 이동하지 않는다

### AC-WALL-2: 상단 경계 충돌
- **Given** 뱀의 머리가 게임판 상단 끝 행에 있고 위쪽으로 진행 중이다
- **When** 한 틱이 진행된다
- **Then** 게임 상태가 `gameover`로 전환된다

---

## 4. 자체 충돌 (Self Collision) — REQ-GAME-003

### AC-SELF-1: 자기 몸과 충돌
- **Given** 게임 상태가 `playing`이고 뱀이 머리부터 꼬리 순서로 다음 세그먼트 좌표를 점유한다: 머리 (5, 5), (4, 5), (4, 6), (5, 6), (6, 6), (6, 5). 진행 방향은 아래쪽(y 증가)이며 음식을 섭취하지 않는다
- **When** 한 틱이 진행되어 머리가 (5, 6)으로 전진한다
- **Then** (5, 6)은 자기 몸 세그먼트(머리에서 세 번째 세그먼트)가 점유한 셀이므로 게임 상태가 `gameover`로 전환된다

### AC-SELF-2: 게임 오버 후 정지
- **Given** 게임 상태가 `gameover`이다
- **When** 추가 틱 호출이 발생한다
- **Then** 뱀은 이동하지 않고 상태는 `gameover`로 유지된다

---

## 5. 점수 증가 (Score Increment) — REQ-GAME-005

### AC-SCORE-1: 섭취 시 증가
- **Given** 현재 점수가 0이다
- **When** 음식을 1회 섭취한다
- **Then** 점수가 `SCORE_PER_FOOD` 상수값만큼 증가한다

### AC-SCORE-2: 누적 증가
- **Given** 현재 점수가 임의의 값 N이다
- **When** 음식을 연속 3회 섭취한다
- **Then** 점수는 N + (음식당 점수 × 3)이 된다

---

## 6. 180도 반전 방지 (Reversal Prevention) — REQ-GAME-004

### AC-REV-1: 반대 방향 입력 무시
- **Given** 뱀이 오른쪽으로 진행 중이다
- **When** 사용자가 왼쪽 방향 키를 누른다
- **Then** 방향 변경은 무시되고 뱀은 다음 틱에도 오른쪽으로 이동한다

### AC-REV-2: 직각 방향 입력 허용
- **Given** 뱀이 오른쪽으로 진행 중이다
- **When** 사용자가 위쪽 방향 키를 누른다
- **Then** 다음 틱에 뱀은 위쪽으로 이동한다

---

## 7. 재시작 (Restart) — REQ-GAME-005

### AC-RESTART-1: 게임 오버 후 재시작
- **Given** 게임 상태가 `gameover`이고 점수가 N(>0)이다
- **When** 사용자가 재시작을 트리거한다
- **Then** 뱀/음식/점수가 초기 상태로 재설정되고 게임 상태가 `playing`으로 전환된다

### AC-RESTART-2: 시작 트리거
- **Given** 게임 상태가 `ready`이다
- **When** 사용자가 시작을 트리거한다
- **Then** 게임 상태가 `playing`으로 전환되고 틱 루프가 시작된다

---

## 8. 렌더링 및 점수 표시 (Rendering / Score Display) — REQ-GAME-005

### AC-RENDER-1: 매 틱 렌더링 스모크 체크
- **Given** 게임 상태가 `playing`이고 `Renderer`에 스파이/모의(mock) Canvas 2D 컨텍스트가 주입되어 있다
- **When** 한 틱이 진행되고 렌더 루틴이 호출된다
- **Then** 게임판, 뱀의 각 세그먼트, 음식, 현재 점수가 모두 그려진다(즉 컨텍스트의 드로잉 호출이 발생하고 점수 표시 요소가 갱신된다)

### AC-RENDER-2: 점수 표시 갱신(수동 검증)
- **Given** 브라우저에서 게임이 `playing` 상태로 실행 중이다
- **When** 음식을 1회 섭취한다
- **Then** 화면에 표시되는 점수 값이 `SCORE_PER_FOOD`만큼 증가한 값으로 갱신된다(브라우저 육안 확인 — 수동 검증 항목)

## 엣지 케이스 (Edge Cases)

- **EC-1 (빈 셀에만 음식 생성)**: 음식은 뱀의 세그먼트가 점유하지 않은 셀에만 생성되어야 한다. 음식이 뱀 몸 위에 겹쳐 생성되지 않는다.
- **EC-2 (보드 가득 참)**: 뱀이 모든 셀을 점유하여 빈 셀이 없으면, 새 음식을 생성하지 않고 크래시 없이 처리한다(클리어/정지 상태).
- **EC-3 (한 틱 내 빠른 다중 키 입력)**: 한 틱 사이에 여러 방향 키가 빠르게 입력되면, 마지막 유효(180도 반전이 아닌) 입력만 다음 틱에 적용된다. 중간 입력으로 인한 즉시 자체 충돌이 발생하지 않는다.
